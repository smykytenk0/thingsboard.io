TheThingsIndustries is a LoRaWAN network designed for connecting devices using the LoRaWAN stack.
After integrating TheThingsIndustries with ThingsBoard, you can connect, communicate, process, and visualize data from devices in the ThingsBoard IoT platform.

## TheThingsIndustries setup

### Step 1. Register an application

<Steps>
1. Go to the TheThingsIndustries console, open the **Applications** section, and press **Add application**.
2. Fill in the required fields:
   - **Application ID** — e.g. `thingsboard-integration`
3. Press **Create application**.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/tti/tti-create-app.png" alt="Create application in TheThingsIndustries console" />

### Step 2. Add a payload decoder

Devices submit data in binary format. There are two decoding options:

- **TheThingsIndustries decoder** — data is decoded before entering ThingsBoard
- **ThingsBoard converters** — uplink/downlink converters decode data from binary format into JSON

This tutorial uses a hybrid approach: an initial transformation into JSON is performed by the TTI decoder, and then ThingsBoard converters handle further processing.

<Steps>
1. Open the application and go to the **Payload formatters** → **Uplink** tab.
2. Select **Custom Javascript formatter** and paste the decoder function below.
3. Press **Save payload function**.
</Steps>

```js title="TTI payload decoder"
function Decoder(bytes, port) {
  var decoded = { temperature: bytes[0] };
  return decoded;
}
```

**Output JSON:**
```json
{
  "temperature": 2
}
```

<DocImage src="/src/assets/images/user-guide/integrations/tti/tti-create-decoder.png" alt="TTI payload decoder configuration" />

### Step 3. Register an end device

<Steps>
1. In the application, open the **End devices** page and press **Add end device**.
2. Fill in the basic settings:
   - **Device ID** — e.g. `thermostat1`
   - **DevEUI** — unique device identifier
3. Press **Network layer settings** and select the configuration for your device.
4. Press **Application layer settings** and generate an **AppSKey** using the generate button.
5. Press **Add end device** to complete registration.
</Steps>

Create an MQTT integration in the TTI console to get the username and password required by ThingsBoard.

## ThingsBoard setup

### Step 5. Create the uplink data converter

The uplink converter transforms incoming TTI messages into the ThingsBoard message format.
The result must contain `deviceName` and `deviceType` — if the device does not yet exist, ThingsBoard creates it automatically.

**Sample payload received from TheThingsIndustries:**
```json
{
  "end_device_ids": {
    "device_id": "thermostat1",
    "application_ids": {
      "application_id": "thingsboard-integration"
    },
    "dev_eui": "ABABABABABABABAA",
    "join_eui": "0000000000000000",
    "dev_addr": "270000BC"
  },
  "correlation_ids": [
    "as:up:01EFEBYDTA1X51TDGPKC1EYK6N",
    "gs:conn:01ED482WRPY2BABY4TYZV57RJG",
    "gs:uplink:01ED9B93M49J8P3FQXCGGCYTGX",
    "ns:uplink:01ED9B93MH00CTY41A6KF674E4",
    "rpc:/ttn.lorawan.v3.AppAs/SimulateUplink:01EFEBYDS9BGD5A9VVZ6GSNBAV",
    "rpc:/ttn.lorawan.v3.GsNs/HandleUplink:01ED9B93MH293RE2TR3F1WMHEG"
  ],
  "received_at": "2020-08-11T08:59:45.869225403Z",
  "uplink_message": {
    "session_key_id": "BXGsg614fdmYH7efd+fRvA==",
    "f_port": 2,
    "f_cnt": 23787,
    "frm_payload": "AhJF8HTI3khf",
    "decoded_payload": {
      "temperature": 2
    },
    "settings": {
      "data_rate": {}
    },
    "received_at": "2020-08-11T10:08:31.505981496Z"
  }
}
```

The `device_id` is mapped to `deviceName` and `application_id` is mapped to `deviceType`.
The `temperature` value from `decoded_payload` is used as telemetry.

<Aside type="tip">
We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=20 collapsible
var data = decodeToJson(payload);
var deviceName = data.end_device_ids.device_id;
var deviceType = data.end_device_ids.application_ids.application_id;

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    telemetry: {
        temperature: data.uplink_message.decoded_payload.temperature
    }
};

/** Helper functions 'decodeToString' and 'decodeToJson' are already built-in **/

return result;
```
</TabItem>
<TabItem label="JavaScript">
```js maxLines=25 collapsible
var data = decodeToJson(payload);
var deviceName = data.end_device_ids.device_id;
var deviceType = data.end_device_ids.application_ids.application_id;

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    telemetry: {
        temperature: data.uplink_message.decoded_payload.temperature
    }
};

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
    var str = decodeToString(payload);
    var data = JSON.parse(str);
    return data;
}

return result;
```
</TabItem>
</Tabs>

### Step 6. Create the downlink data converter

The downlink converter encodes ThingsBoard commands into TTI-compatible messages.
The output must contain a `downlinks` array with `f_port`, `frm_payload`, and `priority` fields.

**Expected output structure:**
```json
{
  "downlinks": [{
    "f_port": 2,
    "frm_payload": "vu8=",
    "priority": "NORMAL"
  }]
}
```

The result object fields:
- `contentType` — how data is encoded: `TEXT`, `JSON`, or `BINARY`
- `data` — the actual payload sent to the device in TTI
- `metadata` — must include `devId` to identify the target device in TTI

This converter takes the `version` field from the incoming ThingsBoard message and sends it as the payload to device `thermostat1`.

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=20 collapsible
var data = {
    downlinks: [{
        f_port: 2,
        confirmed: false,
        frm_payload: btoa(msg.version),
        priority: "NORMAL"
    }]
};

var result = {
    contentType: "JSON",
    data: JSON.stringify(data),
    metadata: {
        devId: 'thermostat1'
    }
};

return result;
```
</TabItem>
<TabItem label="JavaScript">

### Step 7. Create the TTI integration

<Steps>
1. Go to **Integration center** → **Integrations** and click **+** to add a new integration.
2. Select **TheThingsIndustries** as the integration type.
3. Enter a name, e.g. `TTI Integration`.
4. Select the uplink and downlink converters created above.
5. Fill in the connection settings:
   - **Region** — the region where your TTI application is registered (e.g. `eu1`)
   - **Username** — the username from the TTI MQTT integration (e.g. `thingsboard-integration@thingsboard`)
   - **Password** — the password from the TTI MQTT integration
6. Click **Add** to save the integration.
</Steps>

## Validation

### Validate uplink messages

When a device sends data, verify it appears in ThingsBoard:

<Steps>
1. Go to **Devices** → **All** → **thermostat1**.
</Steps>

You will see that:
- A new device named `thermostat1` was automatically registered in ThingsBoard.
- In the **Latest Telemetry** tab the last submitted temperature equals `2`.

<DocImage src="/src/assets/images/user-guide/integrations/tti/tb-device-telemetry.png" alt="ThingsBoard device latest telemetry showing temperature = 2" />

### Validate downlink messages

To test downlink, update the Root Rule Chain to send a downlink message when a device attribute is changed.

<Steps>
1. Open **Rule Chains** → **Root Rule Chain** and click the edit icon.
2. Add an **Integration Downlink** action node.
3. Connect it to the **Message Type Switch** node using the **Attributes Updated** relation.
4. Save the rule chain.
</Steps>

<ImageGallery images={[
  { src: '/src/assets/images/user-guide/integrations/tti/tb-add-rule-downlink.png', alt: 'Add Integration Downlink node to Root Rule Chain', caption: 'Add and configure the Integration Downlink action node' },
  { src: '/src/assets/images/user-guide/integrations/tti/tb-route-to-downlink.png', alt: 'Connect Message Type Switch to Integration Downlink via Attributes Updated', caption: 'Connect via Attributes Updated relation' },
]} />

Now trigger a downlink by adding a shared attribute:

<Steps>
1. Go to **Devices** → **All** → **thermostat1** → **Attributes** tab.
2. Switch to **Shared attributes** and click **+** to add a new attribute.
3. Enter the key `version` and value `v.0.11`, then click **Add**.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/tti/tb-add-version.png" alt="Add shared attribute version = v.0.11 to trigger downlink" />

This triggers a downlink message to device `thermostat1` containing the `version` field value.
Open the TTI console, navigate to the `thingsboard-integration` application, and check the **Data** section —
the downlink message will appear as received (displayed as bytes `76 2E 30 2E 31 31`).

<DocImage src="/src/assets/images/user-guide/integrations/tti/ttn-downlink-verified.png" alt="TTI console confirming downlink message was received by the device" />
