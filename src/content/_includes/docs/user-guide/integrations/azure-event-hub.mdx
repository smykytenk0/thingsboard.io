## Create IoT Hub in Azure

### Step 1. Create the IoT Hub resource

<Steps>
1. Sign in to the <a href="https://portal.azure.com" target="_blank" rel="noopener noreferrer">Azure portal</a> and click **Create a resource**.
2. Search for **IoT Hub**, select it from the list, and click **Create**.
3. Click **Create new** to create a new resource group. Enter a resource group name and an IoT hub name.
4. Click **Review + create**, then click **Create**.
5. Wait for the deployment to complete, then click **Go to resource**.
</Steps>

## ThingsBoard setup

### Step 3. Create the uplink data converter

The uplink converter transforms incoming Azure Event Hub messages into the ThingsBoard message format.

**Sample payload sent from the Azure IoT Hub rule node:**

```json
{
  "devName": "TB-D-01",
  "msg": {
    "temp": 42,
    "humidity": 77
  }
}
```

**The decoder function used in this tutorial:**

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=15 collapsible
var data = decodeToJson(payload);

var deviceType = 'azure-event-hub-device';

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
```

<TabItem label="JavaScript">
```js maxLines=20 collapsible
var data = decodeToJson(payload);
var deviceName = data.devName;
var deviceType = 'azure-event-hub-device';

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    telemetry: {
        temp: data.msg.temp,
        humidity: data.msg.humidity
    }
};

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
    var str = decodeToString(payload);
    var data = JSON.parse(str);
    return data;
}

return result;
```
</TabItem>
</Tabs>

### Step 4. Create the Azure Event Hub integration

<Steps>
  1. Go to **Integration center &#8702; Integrations** and click **+ Add integration**.
  2. **Basic settings:**
      - Set **Integration type** to **Azure Event Hub** and enter a name.
      - Click **Next**.
  3. **Uplink data converter:** select the previously created uplink converter and click **Next**.
  4. **Downlink data converter:** leave empty for now and click **Skip**.
  5. **Connection settings** — fill in the following fields (details below):
      - **Connection String** — the Event Hub-compatible endpoint from your IoT Hub
      - **Storage connection string** — the connection string from your Azure Storage account
      - **Container name** — the name of your Azure Storage container
      - Optionally enable **Persistent checkpoints** to resume from the last processed event on restart
  6. Click **Check connection** to verify, then click **Add**.
</Steps>

## Test the integration

To verify the integration, send a test message from Azure IoT Hub to ThingsBoard using a rule chain with a **Generator** node and an **Azure IoT Hub** node.

### Step 5. Configure a test rule chain

<Steps>
1. Open **Rule Chains** and create a new rule chain (or open an existing one for testing).
2. Add a **Generator** node. Configure it with a name, message count, and the following generate function:
</Steps>

```js title="Generator function"
var msg = {
    devName: "TB-D-01",
    msg: {
        temp: 42,
        humidity: 77
    }
};
var metadata = { data: 40 };
var msgType = "POST_TELEMETRY_REQUEST";

return { msg: msg, metadata: metadata, msgType: msgType };
```

<Steps>
  <ol start="3">
    <li>
      Add an **Azure IoT Hub** node with the following settings:
      <ul>
        <li><strong>Topic</strong> — replace <code>&lt;device_id&gt;</code> with the Device Name (e.g. <code>TB-D-01</code>)</li>
        <li><strong>Hostname</strong> — your IoT Hub hostname from the Azure portal</li>
        <li><strong>Device ID</strong> — the Device Name (e.g. <code>TB-D-01</code>)</li>
        <li><strong>SAS Key</strong> — the device <strong>Primary key</strong> from Azure IoT Hub (see below)</li>
      </ul>
    </li>
    <li>Connect the <strong>Generator</strong> node to the <strong>Azure IoT Hub</strong> node using the <strong>Success</strong> relation.</li>
    <li>Save the rule chain.</li>
  </ol>
</Steps>

#### Find the device Primary key

<Steps>
1. In your IoT Hub resource, open **IoT devices** and click on the device (`TB-D-01`).
2. Copy the **Primary key** value.
</Steps>

### Step 9. Assign the downlink converter to the integration

<Steps>
1. Go to **Integration center &#8702; Integrations** and open the Azure Event Hub integration.
2. Click the edit (pencil) icon.
3. In the **Downlink data converter** field, select the downlink converter you just imported.
4. In the **Advanced settings**, enter the **Event Hub-compatible name** from Step 7 in the **IoT Hub name** field.
5. Click **Apply changes**.
</Steps>

## Test downlink messages

### Step 10. Configure the rule chain for downlinks

<Steps>
1. Find the Rule Chain associated with the device profile used by `TB-D-01`.
2. Open the rule chain and add an **Integration Downlink** action node. Select the Azure Event Hub integration in the node settings.
3. Connect the **Message Type Switch** node to the **Integration Downlink** node using the **Attributes Updated** relation.
4. Save the rule chain.
</Steps>

### Step 11. Trigger a downlink via shared attribute

<Steps>
1. Go to **Entities &#8702; Devices** and open the device `TB-D-01`.
2. Open the **Attributes** tab, switch to **Shared attributes**, and click **+**.
3. Enter an attribute key and value, then click **Add**.
