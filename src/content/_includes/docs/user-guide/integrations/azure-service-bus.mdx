Azure Service Bus Integration allows you to stream data from Azure Service Bus to ThingsBoard and convert device payloads to the ThingsBoard format.

<DocImage src="/src/assets/images/user-guide/integrations/azure-service-bus/azure-service-bus-overview.png" alt="Azure Service Bus integration overview diagram" />

## Configure Azure Service Bus

Before creating the integration in ThingsBoard, set up your Azure Service Bus namespace:

<Steps>
1. Sign in to the <a href="https://portal.azure.com" target="_blank" rel="noopener noreferrer">Azure portal</a> and open your Service Bus namespace (or create one).
2. Create a **topic** for uplink messages and a **subscription** on that topic for ThingsBoard to consume.
3. Copy the **connection string** for the namespace — you will need it when configuring the integration. It is found under **Settings &#8702; Shared access policies** in the namespace.
</Steps>

## ThingsBoard setup

### Step 1. Create the uplink data converter

The uplink converter transforms incoming Azure Service Bus messages into the ThingsBoard message format.

**Sample input message:**

```json
{
  "devName": "Sensor A1",
  "msg": {
    "temp": 23,
    "humidity": 40
  }
}
```

The `devName` field maps to `deviceName`. The nested `msg` object provides `temperature` and `humidity` telemetry.

```js maxLines=20 collapsible
var data = decodeToJson(payload);
var deviceName = data.devName;
var deviceType = 'thermostat';

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    telemetry: {
        temperature: data.msg.temp,
        humidity: data.msg.humidity
    }
};

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
    var str = decodeToString(payload);
    var data = JSON.parse(str);
    return data;
}

return result;
```
</TabItem>

### Step 2. Create the Azure Service Bus integration

<Steps>
  1. Go to **Integration center &#8702; Integrations** and click **+ Add integration**.
  2. **Basic settings:**
      - Set **Integration type** to **Azure Service Bus**.
      - Click **Next**.
  3. **Uplink data converter:** select the previously created uplink converter and click **Next**.
  4. **Downlink data converter:** leave empty for now and click **Skip**.
  5. **Connection settings:**
      - **Connection String** — the namespace connection string from Azure (found under **Settings &#8702; Shared access policies** in your Service Bus namespace)
      - **Topic Name** — the name of the topic you created for uplink messages
      - **Subscription Name** — the name of the subscription on that topic
  6. Optionally click **Check connection** to verify connectivity.
  7. Click **Add** to save the integration.
</Steps>

## Send uplink message

To test the integration, send a message to your Azure Service Bus topic using the **Service Bus Explorer** in the Azure portal:

<Steps>
1. In the Azure portal, open your Service Bus namespace and navigate to the topic.
2. Click **Service Bus Explorer**.
3. Click **Send message**, select **JSON** as the content type, and paste the payload below.
4. Click **Send**.
</Steps>

```json
{
  "devName": "Sensor A1",
  "msg": {
    "temp": 23,
    "humidity": 40
  }
}
```

## Add a downlink converter

To send messages from ThingsBoard back to Azure Service Bus, create a dedicated downlink topic in Azure and add a downlink converter to the integration.

### Step 3. Create the downlink converter

The downlink converter output must include:
- `contentType` — how data is encoded: `TEXT`, `JSON`, or `BINARY`
- `data` — the payload to write to the Service Bus topic
- `metadata.deviceId` — identifier used to target the correct device

### Step 4. Assign the downlink converter and configure the downlink topic

<Steps>
1. Go to **Integration center &#8702; Integrations** and open the Azure Service Bus integration.
2. Click the edit (pencil) icon.
3. In the **Downlink data converter** field, select the downlink converter you just imported.
4. Open **Advanced settings** and fill in:
    - **Downlink topic** — the name of the Azure Service Bus topic dedicated for downlink messages
    - **Downlink connection string** — the namespace connection string (copy from the uplink settings if you use the same namespace)
5. Click **Apply changes**.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/azure-service-bus/azure-service-bus-integration-add-downlink-converter-1.png" alt="Integration edit mode — downlink converter and advanced settings with downlink topic and connection string" />

## Configure the rule chain for downlinks

### Step 5. Find the device's rule chain

<Steps>
1. Go to **Entities &#8702; Devices**, find device `Sensor A1`, and note its **Device profile**.
2. Open **Profiles &#8702; Device profiles** and find the profile used by `Sensor A1`.
3. Note the **Rule chain** assigned to that profile — open it for editing.
</Steps>

<ImageGallery images={[
  { src: '/src/assets/images/user-guide/integrations/azure-service-bus/azure-service-bus-integration-find-device-profile-1.png', alt: 'Devices list showing device profile column for Sensor A1', caption: 'Find the device profile for Sensor A1' },
  { src: '/src/assets/images/user-guide/integrations/azure-service-bus/azure-service-bus-integration-find-device-profile-2.png', alt: 'Device profile details showing assigned rule chain', caption: 'Note the rule chain assigned to the device profile' },

### Step 6. Add the Integration Downlink node

<Steps>
1. Open the rule chain and add an **Integration Downlink** action node. Select the Azure Service Bus integration.
2. Connect the **Message Type Switch** node to the **Integration Downlink** node using the **Attributes Updated** relation.
3. Save the rule chain.
</Steps>

<ImageGallery images={[
  { src: '/src/assets/images/user-guide/integrations/azure-service-bus/azure-service-bus-integration-downlink-rule-chain-1.png', alt: 'Rule chain — Integration Downlink node added and configured', caption: 'Add the Integration Downlink node and select the Azure Service Bus integration' },
  { src: '/src/assets/images/user-guide/integrations/azure-service-bus/azure-service-bus-integration-downlink-rule-chain-2.png', alt: 'Rule chain — Message Type Switch connected to Integration Downlink via Attributes Updated', caption: 'Connect Message Type Switch → Attributes Updated → Integration Downlink' },
]} />

### Step 7. Trigger a downlink via shared attribute

<Steps>
1. Go to **Entities &#8702; Devices** and open device `Sensor A1`.
2. Open the **Attributes** tab, switch to **Shared attributes**, and click **+**.
3. Enter an attribute key and value, then click **Add**.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/azure-service-bus/azure-service-bus-integration-add-downlink-attribute-1.png" alt="Adding a shared attribute to Sensor A1 to trigger a downlink message" />

Go to **Integration center &#8702; Integrations**, open the Azure Service Bus integration, and check the **Events** tab for **Downlink** message type entries.
