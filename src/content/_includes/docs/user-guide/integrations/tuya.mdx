Tuya is an IoT platform that enables device manufacturers and solution providers to connect, manage, and control smart devices through the Tuya Cloud. The Tuya Integration in ThingsBoard allows you to:

- Connect Tuya-managed devices to ThingsBoard.
- Receive telemetry and device state updates.
- Send control commands from ThingsBoard to Tuya devices.
- Visualize device data and manage devices centrally.

This guide walks through an end-to-end integration using a Tuya Smart Plug. A virtual Tuya device can also be used for testing.

## Prerequisites

Before starting, ensure you have:

- A ThingsBoard account with permission to create integrations.
- A <a href="https://iot.tuya.com/" target="_blank" rel="noopener noreferrer">Tuya IoT Platform</a> account.
- At least one Tuya device (physical or virtual).
- The Tuya Smart app installed and signed in on your mobile device.

## Configure Tuya cloud

### Step 1. Create a cloud project

<Steps>
1. In the <a href="https://iot.tuya.com/" target="_blank" rel="noopener noreferrer">Tuya IoT Platform</a>, go to **Cloud &#8702; Project Management** and click **Create Cloud Project**.
2. Fill in the required fields in the pop-up window and click **Create**.
3. In the **Authorize API Services** window, complete the settings and click **Authorize**.
4. Note the **Access ID/Client ID** and **Access Secret/Client Secret** — you will need them when configuring the ThingsBoard integration.
</Steps>

### Step 2. Enable the message service

<Steps>
1. Go to the **Message Service** tab of your cloud project.
2. Toggle to enable **Message Service**.
3. In the pop-up, configure **Message Service Type** and **Alert Contact**, then click **Ok**.
</Steps>

### Step 3. Link your Tuya app account

<Steps>
1. Go to **Cloud &#8702; Project Management**, select your project, then navigate to **Devices &#8702; Link App Account**.
2. Click **Add App Account** and select **Tuya Account Authorization**.
3. Scan the QR code with the Smart Life app to authorize.
4. In the pop-up, select **Automatic Link** and click **Ok**. Your devices now appear under the **All Devices** tab.
</Steps>

### Step 4. Create a messaging rule

Enable a messaging rule so the project receives device status uplinks.

<Steps>
1. Go to the **Message Service** tab and click **Create Messaging Rules**.
2. Click **Add Message Filtering Rule**, select **statusReport**, and click **Release Rule**.
3. Enable the rule.
</Steps>

## ThingsBoard setup

### Step 1. Create the uplink data converter

The uplink converter transforms the incoming Tuya device status payload into the ThingsBoard message format.

<Aside type="tip" title="TBEL">
  We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

**The decoder function used in this tutorial:**

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=25 collapsible
/** Decoder **/

// decode payload to JSON
var data = decodeToJson(payload);

var deviceName = data.?bizData.?devId != null ? data.?bizData.?devId : data.?devId;
var deviceType = 'Tuya device';

var telemetry = [];
if (data.status != null) {
    for (var i = 0; i < data.status.length; i++) {
        var res = {};
        var code = data.status[i].code;
        var value = data.status[i].value;
        if (code == "cur_voltage" || code == "cur_power") {
            value = data.status[i].value / 10;
        } else if (code == "cur_current") {
            value = data.status[i].value / 100;
        }
        res[code] = value;
        telemetry.push(res);
    }
} else {
    telemetry = data;
}

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    attributes: {},
    telemetry: telemetry
};

/** Helper functions 'decodeToString' and 'decodeToJson' are already built-in **/

return result;
```
</TabItem>

```js maxLines=35 collapsible
/** Decoder **/

var data = decodeToJson(payload);

var deviceName = data.devId;
var deviceType = 'Tuya device';

var telemetry = [];
if (data.status != null) {
    for (var i = 0; i < data.status.length; i++) {
        var res = {};
        var code = data.status[i].code;
        var value = data.status[i].value;
        if (code == "cur_voltage" || code == "cur_power") {
            value = data.status[i].value / 10;
        } else if (code == "cur_current") {
            value = data.status[i].value / 100;
        }
        res[code] = value;
        telemetry.push(res);
    }
} else {
    telemetry = data;
}

var result = {
   deviceName: deviceName,
   deviceType: deviceType,
   attributes: {},
   telemetry: telemetry
};

/** Helper functions **/

function decodeToString(payload) {
   return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
   var str = decodeToString(payload);
   var data = JSON.parse(str);
   return data;
}

return result;
```

### Step 3. Create the Tuya integration

<Steps>
1. Go to **Integration center &#8702; Integrations** and click **+ Add integration**.
2. Select type **Tuya**, enter a name, and click **Next**.
3. **Uplink data converter:** select the uplink converter you imported and click **Next**.
4. **Downlink data converter:** select the downlink converter you imported and click **Next**.
5. **Connection settings:**
    - **Region** — select the region matching your Tuya account.
    - **Environment** — select **PROD** for a real physical device, or **TEST** for a virtual device.
    - **Access Id** and **Access Key** — paste the Access ID/Client ID and Access Secret/Client Secret from your Tuya cloud project.
6. Click **Add** to save the integration.
</Steps>

## Configure the Root Rule Chain

To enable RPC commands from ThingsBoard to your Tuya device, route **RPC Request to Device** messages through the downlink converter.

<Steps>
1. Go to **Rule chains** and open the **Root Rule Chain**.
2. Find the **Integration Downlink** node, drag it onto the canvas, name it (e.g. `Tuya Downlink`), select the Tuya integration, and click **Add**.
3. Connect the **Message Type Switch** node to the **Integration Downlink** node using the **RPC Request to Device** relation type.
4. Save the rule chain.
</Steps>

## Verify the integration

Disconnect your Smart Plug from power and reconnect it. The device sends an uplink message with telemetry to ThingsBoard. Go to **Entities &#8702; Devices** to see the device provisioned automatically by the integration.

## Tuya Smart Plug dashboard

To visualize Smart Plug data and test RPC commands, import the ready-made dashboard.

<Steps>
1. Download the dashboard file: <a href="/resources/docs/integrations/tuya/tuya_smart_plug_dashboard.json" download="tuya_smart_plug_dashboard.json"><strong>tuya_smart_plug_dashboard.json</strong></a>
2. Go to **Dashboards** and click **+ &#8702; Import dashboard**.
3. Drag the downloaded JSON file into the import window and click **Import**.
4. Open the **Tuya Smart Plug** dashboard.
5. Enter **Edit** mode and update the **Entity Aliases** — set the `smartPlug` alias to your provisioned device.
</Steps>

<Aside type="note">
  Time series data keys may differ between Tuya device models. If the widgets show no data, update the keys in each widget according to your device's documentation.
</Aside>
