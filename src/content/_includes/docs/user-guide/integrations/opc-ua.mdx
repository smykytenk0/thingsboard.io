The **OPC-UA** integration connects ThingsBoard to an OPC-UA server: it polls configured device nodes at regular intervals, decodes node values via an uplink converter, and pushes telemetry and attributes to the platform. In the reverse direction, it encodes Rule Engine messages via a downlink converter and delivers them to OPC-UA nodes as write operations or method calls.

## Prerequisites

Download and install the <a href="https://www.unified-automation.com/downloads/opc-ua-servers.html" target="_blank">OPC UA C++ Demo Server</a> (Windows only).

<Steps>
1. Launch **UA Admin Dialog** and confirm the hostname/IP address and port are configured correctly.
2. Launch **UaCPPServer** — the console window displays the endpoint URLs.
</Steps>

## Uplink converter

The uplink converter decodes incoming OPC-UA node values and maps them to the ThingsBoard data model. For the full decoder function reference, see <DocLink product={props.product} path="user-guide/integrations/uplink-data-converter" target="_blank">Uplink data converter</DocLink>.

The decoder function receives:
- `payload` — JSON object with node values, e.g. `{"temperature": "72.15819999999641"}`
- `metadata` — node metadata: `opcUaNode_name`, `opcUaNode_namespaceIndex`, `opcUaNode_identifier`, `opcUaNode_fqn`, `integrationName`

**Sample payload:**
```json
{
    "temperature": "72.15819999999641"
}
```

**Sample metadata:**
```json
{
    "opcUaNode_namespaceIndex": "3",
    "opcUaNode_name": "AirConditioner_1",
    "integrationName": "OPC-UA Airconditioners",
    "opcUaNode_identifier": "AirConditioner_1",
    "opcUaNode_fqn": "Objects.BuildingAutomation.AirConditioner_1"
}
```

<Aside type="tip">
Use <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes significantly faster than JavaScript inside ThingsBoard.
</Aside>

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=15 collapsible
var data = decodeToJson(payload);
var deviceName = metadata['opcUaNode_name'];
var namespaceIndex = metadata['opcUaNode_namespaceIndex'];
var deviceType = 'airconditioner';

var result = {
   deviceName: deviceName,
   deviceType: deviceType,
   telemetry: {
   },

       namespaceIndex: namespaceIndex

}

   result.telemetry.powerConsumption = toFixed(data.powerConsumption, 2);
}

   result.attributes.state = data.state == '1' ? true : false;
}

/** Helper functions 'decodeToString' and 'decodeToJson' are already built-in **/

return result;
```

function decodeToString(payload) {
   return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
   var str = decodeToString(payload);
   var data = JSON.parse(str);
   return data;
}

return result;

## Downlink converter

The downlink converter encodes RPC commands from ThingsBoard into OPC-UA write operations or method calls. For the full encoder function reference, see <DocLink product={props.product} path="user-guide/integrations/downlink-data-converter">Downlink data converter</DocLink>.

The output of the downlink converter must follow this structure:

```json
[{
    "contentType": "JSON",
    "data": "{\"writeValues\":[],\"callMethods\":[{\"objectId\":\"ns=3;s=AirConditioner_1\",\"methodId\":\"ns=3;s=AirConditioner_1.Stop\",\"args\":[]}]}",
    "metadata": {}
}]
```

The encoder must return an array with one object:
- `contentType` — `JSON`, `TEXT`, or `BINARY`
- `data` — JSON string with two optional arrays:
  - `writeValues` — `{nodeId, value}` objects; <a href="https://documentation.unified-automation.com/uasdkhp/1.4.1/html/_l2_ua_node_ids.html" target="_blank">NodeId format</a>: `ns=<namespaceIndex>;<identifiertype>=<identifier>`
  - `callMethods` — `{objectId, methodId, args}` objects
- `metadata` — must be an empty object for OPC-UA

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=15 collapsible
var data = {
    writeValues: [],
    callMethods: []
};

if (msgType === 'RPC_CALL_FROM_SERVER_TO_DEVICE') {
    if (msg.method === 'setState') {
        var targetMethod = msg.params === 'true' ? 'Start' : 'Stop';
        var writeValue = {
              nodeId: 'ns=' + metadata['cs_namespaceIndex'] +';s=' + metadata['deviceName'],
              value: msg.params
        };
        data.writeValues.push(writeValue);
        var callMethod = {
              objectId: 'ns=' + metadata['cs_namespaceIndex'] +';s=' + metadata['deviceName'],
              methodId: 'ns=' + metadata['cs_namespaceIndex'] +';s=' + metadata['deviceName']+'.'+targetMethod,
              args: []
        };
        data.callMethods.push(callMethod);
    }
}

var result = {
    contentType: "JSON",
    data: JSON.stringify(data),
    metadata: {}
};

return result;
```
</TabItem>

### Connection settings

<h5>Host</h5>

Hostname or IP address of the OPC-UA server.

<h5>Port</h5>

Port number of the OPC-UA server. Default: `49320`.

<h5>Endpoint</h5>

Optional endpoint path appended to the server address when constructing the OPC-UA connection URL.

<h5>Security</h5>

Message security policy for the OPC-UA session:

- `None` — no encryption or signing
- `Basic128Rsa15` — 128-bit symmetric encryption, RSA-15 key exchange
- `Basic256` — 256-bit symmetric encryption
- `Basic256Sha256` — 256-bit symmetric encryption with SHA-256 signing (recommended for production)

<h5>Identity</h5>

Authentication method used when establishing the OPC-UA session:

- `Anonymous` — no credentials required
- `Username` — authenticate with a username and password

<h5>Mapping type</h5>

Determines how ThingsBoard identifies device nodes on the OPC-UA server:

- `Fully Qualified Name` — matches nodes by their full browsed path (e.g. `Objects.BuildingAutomation.AirConditioner_1`)
- `ID` — matches nodes by namespace index and node identifier

<h5>Device Node Pattern</h5>

Regular expression applied to OPC-UA node names or IDs to identify which nodes represent devices. Only nodes matching this pattern are subscribed to. Example: `Channel1\.Device\d+$`.

<h5>Namespace</h5>

OPC-UA namespace index used to scope node identifiers. Only visible when **Mapping type** is set to `ID`.

<h5>Subscription tags</h5>

Defines which child node values ThingsBoard reads from each matched device node. Each entry has:

- **Key** — the field name used in the ThingsBoard message payload
- **Path** — relative path of the child node under the matched device node
- **Required** — when checked, the tag value must be present in every message; missing values cause the message to fail

<h5>Execute remotely</h5>

When enabled, ThingsBoard generates an **Integration key** and **Integration secret** that allow the integration to run as a separate process outside the ThingsBoard cluster — useful when the OPC-UA server is only accessible from a local network.

<h5>Advanced settings</h5>

- **Application name** — OPC-UA client application name sent during session establishment
- **Application uri** — OPC-UA client application URI
- **Scan period in seconds** — how frequently ThingsBoard polls the OPC-UA server for updated node values. Default: `10`
- **Timeout in milliseconds** — connection and request timeout. Default: `5000`
- **Description** — optional text description for the integration
- **Metadata** — key-value pairs injected into every message as `integrationMetadata` in converter scripts
