LORIOT is a LoRaWAN network designed for connecting devices using the LoRaWAN stack.
After integrating LORIOT with ThingsBoard, you can connect, communicate, process, and visualize data from devices in the ThingsBoard IoT platform.

## Register a LORIOT account

<Steps>
1. Go to the <a href="https://loriot.io/" target="_blank" rel="noopener noreferrer">LORIOT website</a> and choose a service package (e.g. Community Public Network Server).
2. Pick your preferred region and country, then complete the registration and log in.
</Steps>

## ThingsBoard setup

### Step 1. Basic settings

<Steps>
1. Sign in to your ThingsBoard account and go to **Integration center &#8702; Integrations**.
2. Click **+ Add integration** and select **Loriot** from the list.
3. Optionally enable **Debug mode** to track events and errors during development.
4. Click **Next**.
</Steps>

<Aside type="note">
  Enabling debug mode stores all debug events in the database. Starting from ThingsBoard 3.9, full debug events are retained only for the first 15 minutes after the integration is created — afterward, only error events are stored.
</Aside>

### Step 2. Create the uplink data converter

The uplink converter transforms incoming LORIOT messages into the ThingsBoard message format.

Starting from ThingsBoard 4.0, you can specify where message fields go (attributes or telemetry) without writing a decoder function manually.

**Sample uplink message from LORIOT:**

```json
{
  "cmd":  "rx",
  "EUI":  "BE7A000000000552",
  "ts":   1470850675433,
  "ack":  false,
  "fcnt": 1,
  "port": 1,
  "data": "2A3F",
  "freq": 868500000,
  "dr":   "SF12 BW125 4/5",
  "rssi": -130,
  "snr":  1.2
}
```

The `EUI` field uniquely identifies the device. The `data` field contains the encoded payload — in this example, `2A` decodes to temperature `42` and `3F` decodes to humidity `63`.

Using the pattern `Device $eui` as the entity name, the device will be created with the name `Device BE7A000000000552`.

**Follow these steps:**

<Steps>
1. Paste the provided decoder function into the Main decoding configuration field:
</Steps>

<Aside type="tip" title="TBEL">
  We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=15 collapsible
var data = decodeToJson(payload);

// Decode hex data field: first byte = temperature, second byte = humidity
var hexData = data.data;
var decoded = {};
if (hexData != null && hexData.length >= 4) {
    decoded.temperature = parseInt(hexData.substring(0, 2), 16);
    decoded.humidity = parseInt(hexData.substring(2, 4), 16);
}

var result = {
    deviceName: 'Device ' + data.EUI,
    deviceType: 'loriot',
    attributes: {
        eui: data.EUI,
        ack: data.ack,
        port: data.port
    },
    telemetry: decoded
};

return result;
```
</TabItem>
<TabItem label="JavaScript">
```js maxLines=15 collapsible
var data = decodeToJson(payload);

// Decode hex data field: first byte = temperature, second byte = humidity
var hexData = data.data;
var decoded = {};
if (hexData != null && hexData.length >= 4) {
    decoded.temperature = parseInt(hexData.substring(0, 2), 16);
    decoded.humidity = parseInt(hexData.substring(2, 4), 16);
}

var result = {
    deviceName: 'Device ' + data.EUI,
    deviceType: 'loriot',
    attributes: {
        eui: data.EUI,
        ack: data.ack,
        port: data.port
    },
    telemetry: decoded
};

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
    var str = decodeToString(payload);
    var data = JSON.parse(str);
    return data;
}

return result;
```
</TabItem>

### Step 4. Connection

There are two ways to connect LORIOT to ThingsBoard.

#### Option A: Configure HTTP push in LORIOT (recommended)

<Steps>
1. Go to the **Applications** page in the LORIOT UI and open your application.
2. Open the **Output** page and click **Add new output**.
3. Select **HTTP Push** as the output type.
4. Copy the **HTTP endpoint URL** from your ThingsBoard integration details (see [finding the HTTP endpoint URL](#send-uplink-message)).
5. Paste the URL into the HTTP Push configuration and click **Add output**.
</Steps>

To reject requests that don't include specific HTTP headers, enable the **Enable security** option:

<Steps>

2. Click **Add** and enter an arbitrary **Header** name and **Value**.
3. Save the changes.
4. Add the same header and value in the LORIOT HTTP Push output configuration using the format `-H "header:value"` (e.g. `-H "authorization:secret"`).
</Steps>

## Send uplink message

To emulate a device message without a physical LORIOT device, use the following `curl` command:

**Find the HTTP endpoint URL:** Go to **Integration center &#8702; Integrations**, open your Loriot integration, and copy the **HTTP endpoint URL**.

<DocImage src="/src/assets/images/user-guide/integrations/loriot/loriot-copy-http-endpoint.png" alt="Copy HTTP endpoint URL from the Loriot integration details" />

**Without security:**

```bash
curl -v -X POST \
  -d '{"EUI":"BE7A000000000552","data":"2A3F","port":1,"cmd":"rx","dr":"SF12 BW125 4/5","snr":1.2,"ack":"false","freq":868500000,"fcnt":1,"rssi":-130,"ts":1613745998000}' \
  $HTTP_ENDPOINT_URL \
  -H "Content-Type:application/json"
```

**With security:**

```bash
curl -v -X POST \
  -d '{"EUI":"BE7A000000000552","data":"2A3F","port":1,"cmd":"rx","dr":"SF12 BW125 4/5","snr":1.2,"ack":"false","freq":868500000,"fcnt":1,"rssi":-130,"ts":1613745998000}' \
  $HTTP_ENDPOINT_URL \
  -H "Content-Type:application/json" \
  -H "$HEADER:$VALUE"
```

After sending the message, go to **Entities &#8702; Devices**. A new device is automatically created and its **Latest telemetry** tab shows the decoded values.

## Create a downlink converter

The downlink converter encodes ThingsBoard commands into LORIOT-compatible messages. Use either TBEL or JavaScript.

**Required metadata fields:**
- `EUI` — the device EUI from LORIOT
- `port` — the LoRaWAN port (1–223)

To find the device EUI in LORIOT, open the **Devices** section and locate the EUI value for your device.

**The encoder used in this tutorial:**

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=20 collapsible
// Encode downlink data from incoming Rule Engine message
// msg        - JSON message payload
// msgType    - message type, e.g. 'ATTRIBUTES_UPDATED'
// metadata   - key-value pairs with additional message data
var result = {
    // downlink data content type: JSON, TEXT or BINARY (base64 format)
    contentType: "TEXT",
    // downlink data
    data: msg.firmware,
    // metadata must include EUI and port
    metadata: {
        "EUI": metadata.deviceName,
        "port": 1
    }
};
return result;
```
</TabItem>
<TabItem label="JavaScript">
```js maxLines=20 collapsible
// Encode downlink data from incoming Rule Engine message
// msg        - JSON message payload
// msgType    - message type, e.g. 'ATTRIBUTES_UPDATED'
// metadata   - key-value pairs with additional message data
var result = {
    // downlink data content type: JSON, TEXT or BINARY (base64 format)
    contentType: "TEXT",
    // downlink data

    // metadata must include EUI and port

        "EUI": metadata.deviceName,
        "port": 1
    }

return result;
```
</TabItem>
</Tabs>

<Aside type="note">
  If your `data` payload is already in hexadecimal format, add `"isHexEncoded": "true"` to the metadata to avoid double hex encoding:

  ```json
  "metadata": {
    "EUI": "BE7A000000000552",
    "port": 1,
    "isHexEncoded": "true"
  }
  ```
</Aside>

## Enable sending downlinks

To send messages from ThingsBoard to LORIOT devices, enable the **Send downlink** option in the integration settings and fill in the required connection fields.

<DocImage src="/src/assets/images/user-guide/integrations/loriot/loriot-add-integration-send-downlink.png" alt="Integration settings with Send downlink enabled and server/application fields filled in" />

Required fields:

- **Server** — the LORIOT server name from the URL (see [finding server and application ID](#finding-server-and-application-id)).
- **Application ID** — the Application ID from your LORIOT project.
- **Application Access Token** — generated in the LORIOT **Access Tokens** section of your application.

To generate the Application Access Token in LORIOT:

<Steps>
1. Open your application in the LORIOT UI.
2. Go to **Access Tokens** and generate a new token.
3. Copy the token and paste it into the **Application Access Token** field in ThingsBoard.
</Steps>

## Configure the rule chain for downlinks

To send a downlink message when a device attribute changes, configure the Root Rule Chain:

<Steps>
1. Open **Rule Chains &#8702; Root Rule Chain** and click the edit icon.
2. Add an **Integration Downlink** action node and connect it to the **Message Type Switch** node using the **Attributes Updated** relation.
3. Save the rule chain.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/loriot/loriot-rule-chain-integration-downlink.png" alt="Root Rule Chain with Integration Downlink node connected via Attributes Updated relation" />

Now trigger a downlink by updating a shared attribute:

<Steps>
1. Go to **Entities &#8702; Devices** and open the device (e.g. `Device BE7A000000000552`).
2. Open the **Attributes** tab, switch to **Shared attributes**, and click **+**.
3. Enter the key `firmware` and a value (e.g. `01052020.v1.1`), then click **Add**.
4. Edit the attribute and change the value (e.g. to `01052020.v1.2`), then click **Save**.
</Steps>
