AWS Kinesis is a managed service for collecting, processing, and analyzing data streams in real time.
After integrating AWS Kinesis with ThingsBoard, you can process and visualize data from Kinesis streams in the ThingsBoard IoT platform.

## AWS Kinesis setup

### Install and configure the AWS CLI

To create and manage Kinesis streams, install the <a href="https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html" target="_blank" rel="noopener noreferrer">AWS CLI</a> and configure it with your credentials.

You need:

- **AWS Access Key ID**
- **AWS Secret Access Key**
- **AWS region name**

Verify the CLI is working:

```bash
aws kinesis help
```

### Kinesis stream data format

AWS Kinesis stores data in Base64 format. ThingsBoard automatically converts Base64-encoded payloads to character strings before passing them to the uplink converter.

### Create demo streams

This tutorial uses two streams: one for incoming device data (uplink) and one for outgoing commands (downlink).

**Create and verify the uplink stream:**

```bash
aws kinesis create-stream --stream-name tb-test-uplink --shard-count 1
```

```bash
aws kinesis describe-stream --stream-name tb-test-uplink
```

**Create and verify the downlink stream:**

```bash
aws kinesis create-stream --stream-name tb-test-downlink --shard-count 1
```

```bash
aws kinesis describe-stream --stream-name tb-test-downlink
```

## ThingsBoard setup

### Step 1. Create the uplink data converter

The uplink converter transforms incoming Kinesis messages into the ThingsBoard message format.

**Sample payload from AWS Kinesis:**

```json
{
  "devName": "kitchen_thermostat",
  "devType": "thermostat",
  "temperature": 22
}
```

The `devName` field maps to `deviceName`, `devType` maps to `deviceType`, and `temperature` is stored as telemetry.

### Step 2. Create the downlink data converter

The downlink converter encodes ThingsBoard commands into Kinesis-compatible messages.

The converter output must include:
- `contentType` — how data is encoded: `TEXT`, `JSON`, or `BINARY`
- `data` — the payload to write to the Kinesis stream
- `metadata.streamName` — the target Kinesis stream
- `metadata.partitionKey` — the partition key for stream routing

**Expected output structure:**

```json
{
  "contentType": "JSON",
  "data": "{\"devName\":\"kitchen_thermostat\",\"version\":\"0.11\"}",
  "metadata": {
    "streamName": "tb-test-downlink",
    "partitionKey": "1234"
  }
}
```

### Step 3. Create the AWS Kinesis integration

<Steps>
  1. Go to **Integration center &#8702; Integrations** and click **+ Add integration**.
  2. **Basic settings:**
      - Set **Integration type** to **AWS Kinesis** and enter a name (e.g. `kinesis_integration`).
      - **Enable integration** and **Allow create devices or assets** are on by default.
      - Click **Next**.
  3. **Uplink data converter:** select the previously created uplink converter and click **Next**.
  4. **Downlink data converter:** select the previously created downlink converter and click **Next**.
  5. **Connection settings:**
      - **Stream name** — `tb-test-uplink`
      - **Region** — your AWS region (e.g. `us-east-1`)
      - **Access Key ID** — your AWS access key ID
      - **Secret Access Key** — your AWS secret access key
      - **Use credentials from EC2 Instance Metadata Service** — leave disabled unless running on EC2
      - **Use Consumers with Enhanced Fan-Out** — leave disabled for this tutorial
  6. Click **Add** to save the integration.
</Steps>

## Validation

### Validate uplink messages

Send a test record to the uplink stream using the AWS CLI:

```bash
aws kinesis put-record \
  --stream-name tb-test-uplink \
  --partition-key 123 \
  --data '{"devName": "kitchen_thermostat", "devType": "thermostat", "temperature": 22}'
```

Then go to **Entities &#8702; Devices**. The device `kitchen_thermostat` is created automatically. Open it and check the **Latest telemetry** tab — the `temperature` value should equal `22`.

<DocImage src="/src/assets/images/user-guide/integrations/aws-kinesis/aws-kinesis-validate-telemetry.png" alt="Device kitchen_thermostat latest telemetry showing temperature = 22" />

### Validate downlink messages

To send a message from ThingsBoard to the Kinesis downlink stream, configure the Root Rule Chain to trigger a downlink when a device attribute is updated.

<Steps>
1. Open **Rule Chains &#8702; Root Rule Chain** and click the edit icon.
2. Add an **Integration Downlink** action node.
3. Connect it to the **Message Type Switch** node using the **Attributes Updated** relation.
4. Save the rule chain.
</Steps>

<ImageGallery images={[
  { src: '/src/assets/images/user-guide/integrations/aws-kinesis/aws-kinesis-downlink-node.png', alt: 'Integration Downlink node added to the Root Rule Chain', caption: 'Add the Integration Downlink action node' },
  { src: '/src/assets/images/user-guide/integrations/aws-kinesis/aws-kinesis-node-connected.png', alt: 'Integration Downlink node connected to Message Type Switch via Attributes Updated relation', caption: 'Connect via the Attributes Updated relation' },
]} />

Now trigger the downlink by adding a shared attribute to the device:

<Steps>
1. Go to **Entities &#8702; Devices** and open `kitchen_thermostat`.
2. Open the **Attributes** tab, switch to **Shared attributes**, and click **+**.
3. Enter the key `version` and value `v.0.11`, then click **Add**.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/aws-kinesis/aws-kinesis-shared-attr-updated.png" alt="Shared attribute version = v.0.11 added to kitchen_thermostat" />

To verify the message was written to the Kinesis downlink stream, get a shard iterator and retrieve records using the AWS CLI:

```bash
aws kinesis get-shard-iterator \
  --shard-id shardId-000000000000 \
  --shard-iterator-type TRIM_HORIZON \
  --stream-name tb-test-downlink
```

Use the returned `ShardIterator` value in the following command:

```bash
aws kinesis get-records --shard-iterator <ShardIterator>
```
