The Things Stack Community (TTS) is a LoRaWAN network designed for connecting devices using the LoRaWAN stack.
After integrating The Things Stack with ThingsBoard, you can connect, communicate, process, and visualize data from devices in the ThingsBoard IoT platform.

## The Things Stack Community setup

### Step 1. Register an application

<Steps>
1. Go to the <a href="https://console.thethingsnetwork.org/" target="_blank">TTS console</a>, open the **Applications** section, and press **Add application**.
2. Fill in the required fields:
   - **Application ID** — e.g. `thingsboard-connection`
   - **Handler registration** — identifies the region where the application will be registered (e.g. `eu1` for the EU region)
3. Press **Create application**.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/ttn/ttn-add-application.png" alt="Add application in The Things Stack console" />

### Step 2. Add a payload decoder

This tutorial uses a hybrid approach: an initial transformation into JSON is performed by the TTS decoder, and then ThingsBoard uplink/downlink converters handle further processing.

<Steps>
1. Open the application you created and go to the **Payload formatters** → **Uplink** tab.
2. Select **Custom Javascript formatter** and paste the decoder function below.
3. Press **Save payload function**.
</Steps>

```js title="TTS payload decoder"
function Decoder(bytes, port) {
  var decoded = { temperature: bytes[0] };
  return decoded;
}
```

<DocImage src="/src/assets/images/user-guide/integrations/ttn/ttn-decoder.png" alt="TTS payload decoder configuration" />

### Step 3. Register a device

<Steps>
1. In the application, open the **Devices** page and press **Register device**.
2. Fill in the required fields:
   - **Device ID** — e.g. `thermostat-a`
   - **Device EUI** — press **Generate** to create a random identifier
   - **AppEUI** — can be filled with zeros
   - **AppKey** — press **Generate** to create a random identifier
3. Press **Register**.
</Steps>

### Step 4. Test the payload formatter

After the device is added, verify the decoder works correctly.

<Steps>
1. Open the device **thermostat-a** and select the **Payload formatters** tab.
2. In the **Simulate uplink** section, enter `0F` as the payload.
3. Press **Test** and verify the output JSON.
</Steps>

**Input payload (hex):** `0F`

**Expected output:**
```json
{
  "temperature": 15
}
```

<DocImage src="/src/assets/images/user-guide/integrations/ttn/payload_format.png" alt="Payload formatter test showing temperature decoded from hex 0F to 15" />

### Step 5. Generate an API access key

An access key is required to authenticate the ThingsBoard integration with TTS.

<Steps>
1. In the TTS console, go to **API keys** in your application menu.
2. Press **Add API key**, enter a name, and select the required permissions.
3. Press **Generate key** and **copy the key immediately** — it will not be shown again.
</Steps>

## ThingsBoard setup

### Step 6. Create the uplink data converter

The uplink converter transforms incoming TTS messages into the ThingsBoard message format. The converter must produce an object containing `deviceName` and `deviceType`. If the device does not yet exist, ThingsBoard creates it automatically.

**Sample payload received from TTS:**
```json
{
  "end_device_ids": {
    "device_id": "thermostat-a",
    "application_ids": {
      "application_id": "thingsboard-connection"
    },
    "dev_eui": "70B3D57ED00550F2",
    "join_eui": "0000000000000000"
  },
  "correlation_ids": [
    "as:up:01GC9S4G55D3AJ2PG32TCAZ6H1",
    "rpc:/ttn.lorawan.v3.AppAs/SimulateUplink:48ff02a3-cc7d-4097-b301-1411dbae3ca2"
  ],
  "received_at": "2022-09-06T16:11:35.461454886Z",
  "uplink_message": {
    "f_port": 1,
    "frm_payload": "Dw==",
    "decoded_payload": {
      "temperature": 15
    },
    "rx_metadata": [{
      "gateway_ids": {
        "gateway_id": "test"
      },
      "rssi": 42,
      "channel_rssi": 42,
      "snr": 4.2
    }],
    "settings": {
      "data_rate": {
        "lora": {
          "bandwidth": 125000,
          "spreading_factor": 7
        }
      }
    }
  },
  "simulated": true
}
```

The `device_id` is mapped to `deviceName` and `application_id` is mapped to `deviceType`. The `temperature` value from `decoded_payload` is used as telemetry.

<Aside type="tip">
We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=20 collapsible
var data = decodeToJson(payload);
var deviceName = data.end_device_ids.device_id;
var deviceType = data.end_device_ids.application_ids.application_id;

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    telemetry: {
        temperature: data.uplink_message.decoded_payload.temperature
    }
};

/** Helper functions 'decodeToString' and 'decodeToJson' are already built-in **/

return result;
```
</TabItem>
<TabItem label="JavaScript">
```js maxLines=25 collapsible
var data = decodeToJson(payload);
var deviceName = data.end_device_ids.device_id;
var deviceType = data.end_device_ids.application_ids.application_id;

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    telemetry: {
        temperature: data.uplink_message.decoded_payload.temperature
    }
};

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
    var str = decodeToString(payload);
    var data = JSON.parse(str);
    return data;
}

return result;
```
</TabItem>
</Tabs>

### Step 7. Create the downlink data converter

The downlink converter encodes ThingsBoard commands into TTS-compatible messages. The output must contain a `downlinks` array with `f_port`, `frm_payload`, and `priority` fields.

**Expected output structure:**
```json
{
  "downlinks": [{
    "f_port": 2,
    "frm_payload": "vu8=",
    "priority": "NORMAL"
  }]
}
```

The result object must follow this format:
- `contentType` — how data is encoded: `TEXT`, `JSON`, or `BINARY`
- `data` — the actual payload sent to the device. See the <a href="https://www.thethingsnetwork.org/docs/applications/mqtt/api" target="_blank">TTS API documentation</a> for details.
- `metadata` — must include `devId` to identify the target device in TTS

This converter takes the `version` field from the incoming ThingsBoard message and sends it as the payload to device `thermostat-a`.

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=20 collapsible
var data = {
    downlinks: [{
        f_port: 2,
        confirmed: false,
        frm_payload: btoa(msg.version),
        priority: "NORMAL"
    }]
};

var result = {
    contentType: "JSON",
    data: JSON.stringify(data),
    metadata: {
        devId: 'thermostat-a'
    }
};

return result;
```
</TabItem>
<TabItem label="JavaScript">

### Step 8. Create the TTS integration

<Steps>
1. Go to **Integration center** → **Integrations** and click **+** to add a new integration.
2. Select **The Things Stack Community** as the integration type.
3. Select the uplink and downlink converters created above.
4. Fill in the connection settings:
   - **Region** — the region where your TTS application is registered (e.g. `eu1`)
   - **Application ID** — the Application ID from TTS (e.g. `thingsboard-connection`)
   - **Access Key** — the API key generated in Step 5
   - **Use API v3** — set to **Enabled**
5. Optionally click **Check connection** to verify connectivity.
6. Click **Add** to save the integration.
</Steps>

## Validation

### Validate uplink messages

<Steps>
1. In the TTS console, go to the device **thermostat-a** and scroll to the **Simulate uplink** section.
2. Enter `0F` (decimal 15) in the payload field and press **Send**.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/ttn/ttn-send-payload.png" alt="TTS console — simulate uplink with payload 0F" />

After sending, go to **Devices** → **All** → **thermostat-a** in ThingsBoard. You will see that a new device was automatically registered, and in the **Latest Telemetry** tab the temperature value equals `15`.

<DocImage src="/src/assets/images/user-guide/integrations/ttn/tb-device-telemetry.png" alt="ThingsBoard device latest telemetry showing temperature = 15" />

### Validate downlink messages

To test downlink, configure the Root Rule Chain to send a downlink message when a device attribute is changed.

<Steps>
1. Open **Rule Chains** → **Root Rule Chain** and click the edit icon.
2. Add an **Integration Downlink** action node.
3. Connect it to the **Message Type Switch** node using the **Attributes Updated** relation.
4. Save the rule chain.
</Steps>

<ImageGallery images={[
  { src: '/src/assets/images/user-guide/integrations/ttn/tb-add-rule-downlink.png', alt: 'Add Integration Downlink node to Root Rule Chain', caption: 'Add and configure the Integration Downlink action node' },
  { src: '/src/assets/images/user-guide/integrations/ttn/tb-route-to-downlink.png', alt: 'Connect Message Type Switch to Integration Downlink', caption: 'Connect via Attributes Updated relation' },
]} />

Now trigger a downlink message by adding a shared attribute:

<Steps>
1. Go to **Devices** → **All** → **thermostat-a** → **Attributes** tab.
2. Switch to **Shared attributes** and click **+** to add a new attribute.
3. Enter the key `version` and value `v.0.11`, then click **Add**.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/ttn/tb-add-version.png" alt="Add shared attribute version = v.0.11 to trigger downlink" />

This triggers the downlink message to the device `thermostat-a` containing the `version` field value. Open the TTS console, navigate to the application, and check the **Data** section — the downlink message should appear as received.

<DocImage src="/src/assets/images/user-guide/integrations/ttn/ttn-downlink-verified.png" alt="TTS console confirming downlink message was received by the device" />
