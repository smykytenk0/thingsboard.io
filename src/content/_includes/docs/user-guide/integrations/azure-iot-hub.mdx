Azure IoT Hub Integration allows you to stream data from Azure IoT Hub to ThingsBoard and convert device payloads to the ThingsBoard format.

<DocImage src="/src/assets/images/user-guide/integrations/azure-iot-hub/iot-hub-integration.svg" alt="Azure IoT Hub integration overview diagram" />

## Prerequisites

Before creating the integration in ThingsBoard, set up your Azure IoT Hub and register a device:

<Steps>
1. <a href="https://docs.microsoft.com/en-us/azure/iot-hub/iot-hub-create-through-portal#create-an-iot-hub" target="_blank" rel="noopener noreferrer">Create an IoT hub</a> in the Azure portal.
2. <a href="https://docs.microsoft.com/en-us/azure/iot-hub/iot-hub-create-through-portal#register-a-new-device-in-the-iot-hub" target="_blank" rel="noopener noreferrer">Register a new device</a> in the IoT hub. Note the device ID and SAS key — you will need them when configuring the integration.
</Steps>

## ThingsBoard setup

### Step 1. Create uplink data converter

The uplink converter transforms incoming Azure IoT Hub messages into the ThingsBoard message format.

The decoder function receives:
- `payload` — the raw message body, e.g. `{"devName": "T1", "msg": {"temp": 42, "humidity": 77}}`
- `metadata` — integration metadata

**Sample payload:**
```json
{
  "devName": "T1",
  "msg": {
    "temp": 42,
    "humidity": 77
  }
}
```

The `devName` field maps to `deviceName`. The nested `msg` object provides `temperature` and `humidity` telemetry.

    deviceName: deviceName,
    deviceType: deviceType,
    telemetry: {
        temperature: data.msg.temp,
        humidity: data.msg.humidity
    }

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
    var str = decodeToString(payload);
    var data = JSON.parse(str);
    return data;
}

return result;

#### Credentials

Two authentication types are supported:

**Shared Access Signature:**
- **SAS Key** — the device SAS key from Azure IoT Hub. See <a href="https://docs.microsoft.com/en-us/azure/iot-edge/how-to-authenticate-downstream-device#symmetric-key-authentication" target="_blank" rel="noopener noreferrer">symmetric key authentication</a>.
- **CA certificate file** — the Baltimore certificate is used by default. See <a href="https://docs.microsoft.com/en-us/azure/iot-hub/iot-hub-mqtt-support#tlsssl-configuration" target="_blank" rel="noopener noreferrer">TLS/SSL configuration</a> for details.

**PEM:**
- **CA certificate file** — the Baltimore certificate is used by default.
- **Certificate file** — your device certificate.
- **Private key file** — the private key for the certificate.
- **Private key password** — optional password for the private key.

For X.509 CA-signed authentication, see the <a href="https://docs.microsoft.com/en-us/azure/iot-edge/how-to-authenticate-downstream-device#x509-ca-signed-authentication" target="_blank" rel="noopener noreferrer">Microsoft documentation</a> and the <a href="https://github.com/Azure/azure-iot-sdk-c/tree/master/tools/CACertificates" target="_blank" rel="noopener noreferrer">CACertificates instructions</a>.

#### Connection settings

<h5>Hostname</h5>

Your Azure IoT Hub hostname, e.g. `tbot.azure-devices.net`.

<h5>Device ID</h5>

The device ID registered in Azure IoT Hub (e.g. `T1`).

<h5>Credentials</h5>

Authentication type for the MQTT connection:

- `Shared Access Signature` — authenticate using a SAS key
- `PEM` — authenticate using X.509 certificates

<h5>Topic filter</h5>

The MQTT topic used to receive cloud-to-device messages. Format: `devices/{deviceId}/messages/devicebound/#`. See <a href="https://docs.microsoft.com/en-us/azure/iot-hub/iot-hub-mqtt-support#receiving-cloud-to-device-messages" target="_blank" rel="noopener noreferrer">IoT Hub MQTT support</a>.

<h5>Execute remotely</h5>

When enabled, ThingsBoard generates an **Integration key** and **Integration secret** that allow the integration to run as a separate process outside the ThingsBoard cluster — useful when the IoT Hub is only accessible from a restricted network.

<h5>Advanced settings</h5>

- **Protocol version** — MQTT protocol version used for the connection. Default: `MQTT 3.1.1`
- **Max bytes in message** — maximum message size in bytes. Default: `32368`
- **Description** — optional text description for the integration
- **Metadata** — key-value pairs injected into every message as `integrationMetadata` in converter scripts

## Validate uplink messages

To test the integration, send a message to your device from the Azure portal:

<Steps>
1. Open your IoT hub in the Azure portal and navigate to the device `T1`.
2. Click **Message to Device** and enter the JSON payload below.
3. Click **Send Message**.
</Steps>

```json
{
  "devName": "T1",
  "msg": {
    "temp": 42,
    "humidity": 77
  }
}
```

<DocImage src="/src/assets/images/user-guide/integrations/azure-iot-hub/iot-hub-send-test-msg.png" alt="Azure IoT Hub portal — send message to device T1" />

Go to **Entities &#8702; Devices** and open device `T1`. In the **Latest Telemetry** tab, you should see `temperature = 42` and `humidity = 77`.

<DocImage src="/src/assets/images/user-guide/integrations/azure-iot-hub/iot-hub-validate-telemetry.png" alt="Device T1 latest telemetry showing temperature = 42 and humidity = 77" />
