Sigfox Integration allows you to stream data from the Sigfox Backend to ThingsBoard and convert binary device payloads to the ThingsBoard format.

<DocImage src="/src/assets/images/user-guide/integrations/sigfox/sigfox-integration.svg" alt="Sigfox integration overview diagram" />

## Prerequisites

This tutorial uses:

- A ThingsBoard Professional Edition instance running locally.
- A Sigfox account with a registered device.

The example device **Sigfox-2216792** publishes `temperature`, `humidity`, `co2`, and `co2Baseline` readings.

## ThingsBoard setup

### Step 1. Create the uplink data converter

The uplink converter transforms the incoming Sigfox payload into the ThingsBoard message format.

**Sample input message from a Sigfox device:**

```json
{
  "device": "BF1327",
  "time": "1661868952",
  "data": "2502af2102462a",
  "seqNumber": "3737"
}
```

The `device` field maps to the device name. The `data` field is a hex-encoded telemetry string where each pair of characters represents a sensor value: `02af` = temperature, `21` = humidity, `0246` = CO₂, `2a` = CO₂ baseline.

<Aside type="tip" title="TBEL">
  We recommend using <DocLink product={props.product} path="user-guide/tbel">TBEL</DocLink> for converter scripts — it executes much more efficiently than JavaScript inside ThingsBoard.
</Aside>

**The decoder function used in this tutorial:**

<Tabs>
<TabItem label="TBEL (Recommended)">
```js maxLines=30 collapsible
/** Decoder **/

// decode payload to string
var json = decodeToJson(payload);
var deviceName = 'Sigfox-' + json.device;
var deviceType = 'Sigfox Airwits CO2';
var groupName = 'UC-0023 Sigfox Airwits CO2';

var attrByte = parseInt(json.data.substring(0, 2), 16);
var autoCalibration = (attrByte & 0x1) === 1 ? "on" : "off";
var zeroPointAdjusted = ((attrByte & 0x2) >> 1) === 1 ? true : false;
var transmitPower = ((attrByte & 0x4) >> 2) === 1 ? "full" : "low";
var powerControl = ((attrByte & 0x8) >> 3) === 1 ? "on" : "off";
var firmwareVersion = attrByte >> 4;

var temperature = parseInt(json.data.substring(2, 6), 16) / 10.0 - 40;
var humidity = parseInt(json.data.substring(6, 8), 16);
var co2 = parseInt(json.data.substring(8, 12), 16);

var co2Baseline = 0;
var co2BaselineN = parseInt(json.data.substring(12, 14), 16);
if (co2BaselineN === 0) {
    co2Baseline = 400;
} else {
    co2Baseline = co2BaselineN * 10;
}

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    groupName: groupName,
    telemetry: {
        ts: json.time + "000",
        values: {
            temperature: toFixed(temperature, 1),
            humidity: humidity,
            co2: co2,
            co2Baseline: co2Baseline
        }
    },
    attributes: {
        sigfox_id: json.device,
        autoCalibration: autoCalibration,
        zeroPointAdjusted: zeroPointAdjusted,
        transmitPower: transmitPower,
        powerControl: powerControl,
        fwVersion: firmwareVersion
    }
};

/** Helper functions 'decodeToString' and 'decodeToJson' are already built-in **/

return result;
```
</TabItem>

```js maxLines=35 collapsible
var autoCalibration = (attrByte & 0x1) === 1 ? "on" : "off";
var zeroPointAdjusted = ((attrByte & 0x2) >> 1) === 1 ? true : false;
var transmitPower = ((attrByte & 0x4) >> 2) === 1 ? "full" : "low";
var powerControl = ((attrByte & 0x8) >> 3) === 1 ? "on" : "off";
var firmwareVersion = attrByte >> 4;

var temperature = parseInt(json.data.substring(2, 6), 16) / 10 - 40;
var humidity = parseInt(json.data.substring(6, 8), 16);
var co2 = parseInt(json.data.substring(8, 12), 16);

var co2Baseline = 0;
var co2BaselineN = parseInt(json.data.substring(12, 14), 16);
if (co2BaselineN === 0) {
    co2Baseline = 400;
} else {
    co2Baseline = co2BaselineN * 10;
}

var result = {
    deviceName: deviceName,
    deviceType: deviceType,
    groupName: groupName,
    telemetry: {
        ts: json.time + "000",
        values: {
            temperature: parseFloat(temperature.toFixed(1)),
            humidity: parseFloat(humidity),
            co2: co2,
            co2Baseline: co2Baseline
        }
    },
    attributes: {
        sigfox_id: json.device,
        autoCalibration: autoCalibration,
        zeroPointAdjusted: zeroPointAdjusted,
        transmitPower: transmitPower,
        powerControl: powerControl,
        fwVersion: firmwareVersion
    }
};

/** Helper functions **/

function decodeToString(payload) {
    return String.fromCharCode.apply(String, payload);
}

function decodeToJson(payload) {
    var str = decodeToString(payload);
    var data = JSON.parse(str);
    return data;
}

return result;
```

### Step 2. Create the SigFox integration

<Steps>
1. Go to **Integration center &#8702; Integrations** and click **+ Add integration**. Select type **SigFox** and enter a name (e.g. `SigFox Integration`). Click **Next**.
2. **Uplink data converter:** select the uplink converter you just imported and click **Next**.
3. **Downlink data converter:** leave empty for now and click **Skip**.
4. **Connection settings:** copy the generated **HTTP endpoint URL** — you will need it when configuring the Sigfox callback. If you want to restrict access, enable **Security** and add a custom header with an arbitrary key and value. Click **Add** to save the integration.
</Steps>

<Aside type="note">
  If **Allow create devices or assets** is unchecked, ThingsBoard will not automatically create a device when a message arrives for an unknown device ID.
</Aside>

## Configure Sigfox backend

Now configure your Sigfox account to forward device messages to ThingsBoard.

### Step 3. Set the downlink mode on your device type

<Steps>
1. In your Sigfox account, go to **Device type** and open your device type for editing (e.g. `thermostats`).
2. In the **Downlink data** section, set the callback downlink mode.
</Steps>

<ImageGallery images={[
  { src: '/src/assets/images/user-guide/integrations/sigfox/sigfox-device-edit-device-type-1.png', alt: 'Sigfox — edit device type', caption: 'Open your device type for editing' },
  { src: '/src/assets/images/user-guide/integrations/sigfox/sigfox-device-edit-device-type-2.png', alt: 'Sigfox — set downlink data callback mode', caption: 'Set the downlink data callback mode' },
]} />

### Step 4. Create a custom callback

<Steps>
1. Go to the **Callbacks** tab of your device type.
2. Click **New** in the upper right corner and select **Custom callback**.
3. Set **Custom payload config**, **Header filter**, and paste the ThingsBoard HTTP endpoint URL into the **URL pattern** field.
4. Enter the message body below and click **Ok**.
</Steps>

<ImageGallery images={[
  { src: '/src/assets/images/user-guide/integrations/sigfox/sigfox-device-callbacks-2.png', alt: 'Sigfox — create new custom callback', caption: 'Click New and select Custom callback' },

**Callback message body:**

```json
{
  "device": "{device}",
  "time": "{time}",
  "data": "{data}",
  "seqNumber": "{seqNumber}"
}
```

### Step 7. Assign the downlink converter to the integration

Go to **Integration center &#8702; Integrations**, open the SigFox integration, click the edit (pencil) icon, select the downlink converter, and click **Apply changes**.

<DocImage src="/src/assets/images/user-guide/integrations/sigfox/sigfox-add-downlink-converter.png" alt="SigFox integration edit mode — downlink converter assigned" />

### Step 8. Configure the Root Rule Chain

<Steps>
1. Go to **Rule chains** and open the **Root Rule Chain**.
2. Find the **Integration Downlink** node, drag it onto the canvas, name it `SigFox Integration Downlink`, select the SigFox integration, and click **Add**.
3. Connect the **Message Type Switch** node to the **Integration Downlink** node using both **Attributes Updated** and **Post Attributes** relation types.
4. Save the rule chain.
</Steps>

### Step 9. Test the downlink

<Steps>
1. Go to **Entities &#8702; Devices**, open your device, and click the **Attributes** tab.
2. Switch to **Shared attributes**, click **+**, enter a key and value, and click **Save**.
3. Send an uplink message from the device to trigger downlink delivery.
</Steps>

<DocImage src="/src/assets/images/user-guide/integrations/sigfox/sigfox-create-shared-attribute.png" alt="Device shared attribute added to trigger downlink" />
