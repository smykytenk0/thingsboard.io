Devices connect directly to ThingsBoard Edge, not to the cloud server. Edge exposes the same transport protocols as the server — MQTT, HTTP, and CoAP — on the local network. This means devices communicate with Edge even when the internet connection is unavailable.

```plaintext
  Device                   ThingsBoard Edge           ThingsBoard Server
    │                             │                          │
    │── MQTT / HTTP / CoAP ──────▶│                          │
    │                             │── telemetry / attrs ────▶│
    │                             │   (when connected)       │
```

## Supported protocols and ports

| Protocol | Default port | TLS port |
|----------|-------------|----------|
| MQTT | 1883 | 8883 |
| HTTP | 8080 | 443 |
| CoAP | 5683 | 5684 |

Use the hostname or IP address of your Edge instance in place of `$EDGE_HOST` in all examples below.

## Get device credentials

Each device authenticates with an access token or an X.509 certificate. Access tokens are the simplest option and work with all three protocols.

<Steps>
1. Log in to your Edge instance and go to **Entities → Devices**.

2. Open the device and click **Copy access token** on the **Device details** page.
</Steps>

Store the token as an environment variable for convenience:

```bash
export ACCESS_TOKEN=your_device_token
export EDGE_HOST=your_edge_host_or_ip
```

## Send telemetry to verify connectivity

Use the protocol your device supports to send a test telemetry message.

<Tabs>
<TabItem label="MQTT">

```bash
mosquitto_pub -d \
  -h $EDGE_HOST \
  -t "v1/devices/me/telemetry" \
  -u "$ACCESS_TOKEN" \
  -m '{"temperature": 25}'
```

</TabItem>
<TabItem label="HTTP">

```bash
curl -v -X POST \
  "http://$EDGE_HOST:8080/api/v1/$ACCESS_TOKEN/telemetry" \
  --header "Content-Type: application/json" \
  --data '{"temperature": 25}'
```

</TabItem>
<TabItem label="CoAP">

```bash
coap post coap://$EDGE_HOST/api/v1/$ACCESS_TOKEN/telemetry \
  --payload '{"temperature": 25}'
```

</TabItem>
</Tabs>

After sending, go to **Entities → Devices**, open the device, and select the **Latest telemetry** tab. The `temperature` key appears with the value you sent.

## TLS connections

To connect over TLS (MQTT port 8883 or HTTPS port 443), the device must trust the Edge instance's certificate. Configure TLS on the Edge transport layer first — see <DocLink product={props.product} path="user-guide/grpc-ssl" target="_blank">gRPC over SSL</DocLink> for certificate management on Edge.

<Aside type="note">
TLS configuration for device-to-Edge transport (MQTT 8883, HTTPS) is separate from the gRPC SSL configuration used for Edge-to-server communication.
</Aside>

## X.509 certificates

For production deployments where access tokens are not acceptable, ThingsBoard supports X.509 certificate-based authentication. Provision the device certificate via **Device credentials** on the **Device details** page.

For the full certificate provisioning reference, see <DocLink product={props.product === Products.EDGE_PE ? Products.PE : Products.CE} path="user-guide/ssl/mqtt-client-mutual-tls" target="_blank">Mutual TLS for MQTT</DocLink>.
