An <a href={props.product === Products.EDGE_PE ? "/docs/pe/user-guide/alarms/" : "/docs/user-guide/alarms/"} target="_blank">alarm</a> in ThingsBoard represents an abnormal condition on a device or asset. Alarms have a type, severity, and lifecycle: they are created when a condition is met and cleared when it is resolved.

In this step, you will connect a **Create alarm** node to the **True** output of the filter added in the previous step, trigger an alarm by sending a high-temperature reading, and then clear it.

---

## Step 4.1 Add a Create alarm node

<Steps>
1. In the Edge UI, go to **Rule chains** and open **Edge Root Rule Chain**.
2. Click **+** and select **Action** &#8702; **Create alarm**.
3. Configure the node:

   | Field | Value |
   |---|---|
   | Name | `Create high temperature alarm` |
   | Alarm type | `High temperature` |
   | Alarm severity | `Critical` |
   | Propagate | Off |

4. Click **Add**.
5. Draw a connection from **High temperature filter** (relation type **True**) → **Create high temperature alarm**.
6. Click **Apply changes**.
</Steps>

---

## Step 4.2 Trigger the alarm

Send a telemetry reading with `temperature` above 30 to activate the alarm. Replace `YOUR_ACCESS_TOKEN` with the device token from Step 2.1.

```bash
curl -X POST http://localhost:8080/api/v1/YOUR_ACCESS_TOKEN/telemetry \
  -H "Content-Type: application/json" \
  -d '{"temperature": 35}'
```

<Steps>
1. In the Edge UI, go to **Entities** &#8702; **Devices** and open `sensor-01`.
2. Open the **Alarms** tab. A **Critical** alarm of type **High temperature** appears.
3. Click the alarm to open its details. Note the **Created time** and **Severity**.
</Steps>

---

## Step 4.3 Acknowledge and clear the alarm

<Steps>
1. In the alarm details, click **Acknowledge**. The alarm status changes to **Acknowledged**.
2. Send a reading below the threshold to simulate the condition resolving:

   ```bash
   curl -X POST http://localhost:8080/api/v1/YOUR_ACCESS_TOKEN/telemetry \
     -H "Content-Type: application/json" \
     -d '{"temperature": 22}'
   ```

3. The alarm status remains **Acknowledged** — ThingsBoard does not auto-clear alarms unless a **Clear alarm** node is added to the rule chain. To clear it manually, click **Clear** in the alarm details.
</Steps>

<Aside type="tip">
To auto-clear the alarm when temperature drops below the threshold, add a **Script** filter node (returning `true` when `temperature &lt;= 30`) and connect it to a **Clear alarm** node with the same alarm type. This is the recommended pattern for production rule chains.
</Aside>
