In this step, you will create a device on Edge, send telemetry to it, and verify the data appears on both Edge and your ThingsBoard account.

---

## Step 2.1 Add a device on Edge

Edge synchronizes devices to your ThingsBoard account automatically.

<Steps>
1. In the Edge UI (`http://localhost:8080`), go to **Entities → Devices** and click **Add Device** (+).

2. Enter a name (like `sensor-01`) and click **Add**.

3. Open the device and go to the **Credentials** tab. Note the **Access token**. You will use it to send telemetry in the next step.
</Steps>

## Step 2.2 Send telemetry

Send a test telemetry message to Edge using HTTP. Replace `YOUR_ACCESS_TOKEN` with the token from Step 2.1.

```bash
curl -X POST http://localhost:8080/api/v1/YOUR_ACCESS_TOKEN/telemetry \
  -H "Content-Type: application/json" \
  -d '{"temperature": 22.5, "humidity": 60}'
```

A `200 OK` response confirms Edge received the data.

<Aside type="tip">
You can also use MQTT or CoAP to publish telemetry. For details, see the <DocLink product={props.product} path="reference/mqtt-api">MQTT API reference</DocLink>.
</Aside>

---

## Step 2.3 View telemetry on Edge

<Steps>
1. In the Edge UI, go to **Entities → Devices** and open `sensor-01`.

2. Open the **Latest telemetry** tab. The `temperature` and `humidity` values appear within a few seconds.
</Steps>

## Step 2.4 Verify data in your ThingsBoard account

Because Edge synchronizes telemetry upstream, the same data is available in your ThingsBoard account.

<Steps>
1. In your ThingsBoard account, go to **Entities → Devices**. The `sensor-01` device appears there — it was synchronized from Edge when you created it in Step 2.1.

2. Open `sensor-01` and go to the **Latest telemetry** tab. The values sent in Step 2.2 are there.
</Steps>

## How the Push to Cloud rule node works

The **Edge Root Rule Chain** forwards telemetry to your ThingsBoard account.

In the Edge UI, go to **Rule Chains** and open **Edge Root Rule Chain**. You will see a **Push to Cloud** rule node connected after the **Save Timeseries** node. This node forwards every telemetry message to your ThingsBoard account once it is stored in the local Edge database.

```plaintext
  Device  ──▶  [Input]  ──▶  [Save Timeseries]  ──▶  [Push to Cloud]  ──▶  Server
                                     │
                                     ▼
                              (local storage)
```

You can modify this Rule Chain to push only specific keys, filter by device type, or add transformations before data leaves Edge.
