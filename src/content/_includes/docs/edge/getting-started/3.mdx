A <a href={props.product === Products.EDGE_PE ? "/docs/pe/user-guide/rule-engine/#rule-chain" : "/docs/user-guide/rule-engine/#rule-chain"} target="_blank">rule chain</a> is a graph of rule nodes that processes incoming messages — telemetry, attributes, RPC calls — and routes them based on conditions. ThingsBoard Edge ships with a default **Edge Root Rule Chain** that saves telemetry locally and forwards it to the cloud.

In this step, you will add a filter node to the root rule chain that detects high-temperature readings and routes them to a separate path — which the next step will use to trigger an alarm.

---

## Step 3.1 Open the Edge Root Rule Chain

<Steps>
1. In the Edge UI (`http://localhost:8080`), go to **Rule chains** and open **Edge Root Rule Chain**.
2. Locate the **Save timeseries** node. Messages flow from it to the **Push to cloud** node, which forwards telemetry to your ThingsBoard account.
</Steps>

The default flow looks like this:

```plaintext
  [Input] ──▶ [Message type switch]
                       │
                  TELEMETRY
                       ▼
             [Save timeseries] ──▶ [Push to cloud] ──▶ Server
```

---

## Step 3.2 Add a script filter node

Add a **Script** node that checks whether the temperature value exceeds the threshold.

<Steps>
1. In the rule chain editor, click **+** and select **Filter** &#8702; **Script**.
2. Enter a name, like `High temperature filter`.
3. Paste the following script into the editor:

   ```javascript
   return typeof msg.temperature !== 'undefined' && msg.temperature > 30;
   ```

4. Click **Add**.
</Steps>

The node returns `True` when `temperature` exceeds 30 and `False` otherwise.

---

## Step 3.3 Connect the filter node

Wire the filter into the existing flow so every telemetry message passes through it after being saved.

<Steps>
1. Remove the connection between **Save timeseries** and **Push to cloud**: hold the connection line and drag it away, then confirm deletion.
2. Draw a connection from **Save timeseries** → **High temperature filter**.
3. Draw a connection from **High temperature filter** (relation type **False**) → **Push to cloud**.
4. Leave the **True** output unconnected for now — the next step will attach an alarm node there.
5. Click **Apply changes** to save the rule chain.
</Steps>

The updated flow:

```plaintext
  [Save timeseries] ──▶ [High temperature filter]
                                  │               │
                                True            False
                                  │               │
                              (next step)   [Push to cloud]
```

<Aside type="note">
All telemetry still reaches the cloud via the **False** path. Only readings above 30 °C will take the **True** path — which the alarm step will use.
</Aside>
