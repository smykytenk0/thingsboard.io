Configure your custom mobile app to launch directly from a <a href={props.product === Products.MOBILE_PE ? '/docs/pe/user-guide/mobile-app-center/qr-code-widget/' : '/docs/user-guide/mobile-app-center/qr-code-widget/'}>QR code scan</a> using the phone's camera.

<Aside type="note">
  This guide applies to app version 1.7.0 and later. For earlier versions, see the pre-1.7.x QR code configuration <a href={props.product === Products.MOBILE_PE ? '/docs/mobile/pe/qr-code-settings-before-v1.7/' : '/docs/mobile/qr-code-settings-before-v1.7/'}>guide</a>.
</Aside>

## iOS application setup

Navigate to **Mobile Center → Applications → Your custom iOS app** in the ThingsBoard platform.

### App Site Association ID

In your app store info settings, provide the **App Site Association ID**. It consists of two parts separated by a dot: `{Team ID}.{Bundle ID}`.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/mobile/qr-code-settings/qr-code-settings-ios-store-info.png', alt: 'App Site Association ID field in app store info settings', caption: 'Enter the App Site Association ID in the app store info settings.' },
]} />

**Team ID** — find it in your <a href="https://developer.apple.com/account" target="_blank" rel="noopener noreferrer">Apple Developer Account</a> under **Membership details**.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/mobile/qr-code-settings/qr-code-settings-team-id.png', alt: 'Team ID location in Apple Developer Account Membership details', caption: 'Find your Team ID under <strong>Membership details</strong> in the Apple Developer Account.' },
]} />

**Bundle ID** — must match the Application package configured in your app settings.

## Android application setup

Navigate to **Mobile Center → Applications → Your custom Android app** in the ThingsBoard platform.

### SHA-256 fingerprint

Provide the **SHA-256 fingerprint** of your app's signing key.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/mobile/qr-code-settings/qr-code-settings-sha256.png', alt: 'SHA-256 fingerprint field in Android app settings', caption: 'Enter the SHA-256 fingerprint in the Android app settings.' },
]} />

- **Google Play distributed apps** — find the SHA-256 certificate fingerprint in the Google Play Console under **Release → Setup → App Integrity → App Signing** tab.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/mobile/qr-code-settings/sha256-fingerprint.png', alt: 'SHA-256 fingerprint location in Google Play Console App Signing tab', caption: 'Find the SHA-256 certificate fingerprint in the Google Play Console under <strong>App Signing</strong>.' },
]} />

- **Locally distributed apps** — use the SHA-256 key that your APK is signed with.

For details, see <a href="https://docs.flutter.dev/cookbook/navigation/set-up-app-links" target="_blank" rel="noopener noreferrer">Set up app links for Android</a> in the Flutter documentation.
