Automatically add or remove a device from an entity group based on incoming telemetry, using the
<DocLink product={props.product} path="user-guide/rule-engine" target="_blank">Rule Engine</DocLink>.
The group is created on the fly when the first matching device is added.

**Example:** Place devices that report `temperature > 30` into a *High temperature* group and remove them
when temperature drops back to `≤ 30`.

## Prerequisites

- A device publishing temperature telemetry — see <DocLink product={props.product} path="user-guide/devices" target="_blank">Devices</DocLink>
- Familiarity with <DocLink product={props.product} path="user-guide/rule-engine" target="_blank">Rule Engine</DocLink> and
  <DocLink product={props.product} path="user-guide/groups" target="_blank">Entity Groups</DocLink>

## Step 1. Create the rule chain

<Steps>
<ol>
  <li>
    <span>Go to <strong>Rule chains</strong> and click <strong>+ Add rule chain &#8702; Create new rule chain</strong>.</span>
    <ul>
      <li>Name it <code>Add Device to Group</code></li>
      <li>Click <strong>Add</strong> and open it.</li>
    </ul>
  </li>

  <li>
    <span>Add a <DocLink product={props.product} path="reference/rule-engine/nodes/filter/script/" target="_blank">script</DocLink> filter node:</span>
    <ul>
      <li>Name it <code>Has temperature</code></li>
      <li>
        <p>Set the script to:</p>
        <Code code={"return msg.containsKey('temperature') && msg.temperature != null;"} lang="javascript" />
      </li>
      <li>Click <strong>Add</strong>.</li>
    </ul>
    <p>This expression returns <code>true</code> if <code>msg</code> contains the <code>temperature</code> field and its value is not <code>null</code>; otherwise, it returns <code>false</code>.</p>
  </li>

  <li>
    <span>Add a second <DocLink product={props.product} path="reference/rule-engine/nodes/filter/script/" target="_blank">script</DocLink> filter node:</span>
    <ul>
      <li>Name it <code>High temperature</code></li>
      <li>
        <p>Set the script to:</p>
        <Code code={'return msg.temperature > 30;'} lang="javascript" />
      </li>
      <li>Click <strong>Add</strong>.</li>
    </ul>
    <p>This expression returns <code>true</code> if the <code>temperature</code> value is greater than <code>30</code>; otherwise, returns <code>false</code>.</p>
  </li>

  <li>
    <span>Add an <DocLink product={props.product} path="reference/rule-engine/nodes/action/add-to-group" target="_blank">add to group</DocLink> action node:</span>
    <ul>
      <li>Name it <code>Add to High Temperature group</code></li>
      <li>
        <p>Specify <strong>group name pattern</strong>:</p>
        <Code code={'High Temperature (${deviceType})'} lang="text" />
      </li>
      <li>Enable <strong>Create new group if not exists</strong></li>
      <li>Click <strong>Add</strong>.</li>
    </ul>
  </li>

  <li>
    <span>Add a <DocLink product={props.product} path="reference/rule-engine/nodes/action/remove-from-group" target="_blank">remove from group</DocLink> action node:</span>
    <ul>
      <li>Name it <code>Remove from High Temperature group</code></li>
      <li>
        <p>Specify <strong>group name pattern</strong>:</p>
        <Code code={'High Temperature (${deviceType})'} lang="text" />
      </li>
      <li>Click <strong>Add</strong>.</li>
    </ul>
  </li>

  <li>
    <span>Connect the nodes:</span>
    <ul>
      <li><strong>Has temperature &#8702;</strong> <code>True</code> <strong>&#8702; High temperature</strong></li>
      <li><strong>High temperature &#8702;</strong> <code>True</code> <strong>&#8702; Add to High temperature group</strong></li>
      <li><strong>High temperature &#8702;</strong> <code>False</code> <strong>&#8702; Remove from High Temperature group</strong></li>
    </ul>
  </li>

  <li>Save the rule chain.</li>
</ol>
</Steps>

Alternatively, you can skip the manual setup and import the ready-made configuration: <a href="/resources/guides/add-devices-to-group/add_device_to_group_rule_chain.json" download="add_device_to_group_rule_chain.json">add_device_to_group_rule_chain.json</a>

## Step 2. Route telemetry to the new rule chain

<Steps>
<ol>
  <li>Open the <strong>Root Rule Chain</strong>.</li>
  <li>
    <span>Add a <strong>Rule Chain</strong> node:</span>
    <ul>
      <li>Name it <code>Add Device to Group</code></li>
      <li>Select <code>Add Device to Group</code> rule chain as the target.</li>
    </ul>
  </li>
  <li>Connect the existing telemetry path (e.g. the <strong>Save Timeseries</strong> node output, or a <strong>Message Type Switch</strong> <code>Post telemetry</code> branch) to this <strong>rule chain</strong> node.</li>
  <li>Apply changes.</li>
</ol>
</Steps>

## Result

When a device publishes telemetry:

| Condition | Outcome |
|-----------|---------|
| `temperature > 30` | Device added to *High temperature \{deviceType\}* group (created automatically if needed) |
| `temperature ≤ 30` | Device removed from that group |

Open the <strong>Devices</strong> page and verify that the device is not assigned to any group (the <strong>Groups</strong> column is empty).
- When a device publishes telemetry <code>temperature &gt; 30</code> &#8702; the device is automatically added to the <strong>High Temperature (thermostat)</strong> group.
- When a device publishes telemetry  <code>temperature ≤ 30</code> &#8702; the device is automatically removed from the <strong>High Temperature</strong> group.
